Reflection:

        This project was really helpful for me. I knew from the beginning of (and way before) this project that I would continue on through Computer Engineering, but it was interesting delving into an aspect of it that I knew I ’d have to do eventually but never really gave too much thought. Technical write ups, reports, and presentations take up a large majority of engineer ’s time and resources but undergraduates usually don ’t start until late junior or senior year as evident by my degree checklist placing technical writing classes at the end of my credit plan. This project brought examples of technical writing to the forefront for me and caused me to look more closely at what I am going to be doing for the rest of my career.

The Yakhno and Ekin paper on the student conference was more of a reference on what technical writing looks like and the content was less of a resource to me, but I still got use out of it. The Roy paper with the case study on a different teaching style was useful both in content and as an example. I ’m kind of stubborn when it comes to my learning style so I probably won ’t actually use any of this information, but it shows me what I could do if I choose to change my ways.

This is my first time properly using and maintaining a Git/GitHub project  which is valuable practice for me. I say properly because I have made many in the past but it was really just a way to store and remotely host things for myself, this is the first time someone other than maybe a friend is looking at one of my GitHub projects so I had to actually make it cohesive and look nice. This is useful because Git is the primary version control software that many businesses use to maintain not only external products but internal tools. In addition, GitHub is not only a widely adopted repository host but a major influencer in the repository hosting category. Many standards that GitHub has set carry over into other repository hosts so skills learned here will be useful even if a future employer uses a different or custom product. The process took a bit of getting used to but in the end I now know my way around the Git terminal program and the owning part of a repository on the GitHub website.

In addition, this is the first time in a long time (like 5 years) that I have written HTML, CSS, and JavaScript. I had gotten to the point where making little websites was trivial and I wanted to move on to compiled languages that give a programmer more freedom and specificity on what they want to do. Doing this project as a website made me brush up on my HTML, CSS, and JavaScript which could be helpful not only in my career moving forward but in future academic projects if I need to make a website again.

In conclusion, I ’m happy with the project I made. I could have started earlier and saved myself from a bit of stress and gave myself a couple hours of sleep, but that ’s my MO and I don ’t learn from mistakes.
