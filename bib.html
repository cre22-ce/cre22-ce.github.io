Bibliography:

    Terminology:

  • git: a distributed version control program that allows multiple collaborators to work on one codebase
  • Github.com: a website that utilizes the git protocol to maintain user repositories
  • repository/repo: a collection of files that represent an entire codebase
  • branch: different versions of the same codebase
  • master branch: the default and main branch in a repository
  • merge: to combine two or more branches into one branch
  • push: to merge individual files from one branch to another
  • pull request: create, modify, and request a merge of a new branch to the master branch ("pulling" the new branch into master)
  • commit: description attatched to a push; sometimes interchangable with push
  • clone: to download a repository
  • fetch: to download a file from a separate repository
  • release: a public distribution of the repository
  • stable release: a stable, debugged, immutable distribution of the repository
  • nightly release: an experimental, sometimes buggy, mutable distribution of the repository
