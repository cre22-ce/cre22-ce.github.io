Bibliography:

    Terminology:

  • git: a distributed version control program that allows multiple collaborators to work on one codebase
  • GitHub.com: a website that utilizes the git protocol to maintain user repositories
  • repository/repo: a collection of files that represent an entire codebase
  • branch: different versions of the same codebase
  • master branch: the default and main branch in a repository
  • issue: a detailed description of a bug report or feature request, or a call for assistance about using some part of the code
  • commit: a group of changes requesting to be pushed to master (usually represented by a new branch)
  • merge: to combine two or more branches or commits into one branch
  • push: to move a local file to a remote location (the repository)
  • pull request: to request a merge of one branch to the master branch ("pulling" the branch into master)
  • clone: to download a repository
  • fetch: to download a file from a separate repository
  • release: a public distribution of the repository
  • stable release: a stable, debugged, immutable distribution of the repository
  • nightly release: an experimental, sometimes buggy, mutable distribution of the repository

  • Research Paper on Writing in Computer Engineering

    (Link to paper)
    •  From Student Conference as a Student Centred Environment for Integrating Technical Writings into Computer Engineering Curriculum, by Yakhno, Ekin.

    This paper serves as two examples of writing in Computer Engineering. Not only is the content about writing in Computer Engineering, but it itself is writing in the Computer Engineering field. This paper by Turkish denizens Professor Yakhno and Assistant Professor Ekin focuses on the reformation of Computer Engineering teaching practices in Turkey, specifically in İzmir University in Karabağlar, İzmir, Turkey. For new Computer Engineering students at İzmir University, technical English and technical writing are given as introductory courses to improve communication skills and increase self confidence, but it can be potentially ineffective to fresh Engineering students. Professor Yakhno and Assistant Professor Ekin show how having a student organized conference with team centered projects can actually do a better job at teaching students better communication skills than the traditional classes by taking a "Project-Based Learning approach."
      This paper is useful to me for the second example of writing: it shows how a proper peer-reviewed journal might look and function. This paper uses various graphics, data, and lists to get its point across to the reader. While I most likely would not be writing about this topic, the formatting is potentially a good template to compare a journal to.

  • Paper on Teaching Technical Writing in Computer Engineering

    (Link to paper)
    •  From Concept Maps for Teaching Technical Writing to Computer Science Majors: A Case Study in a Japanese Technical Institute, by Roy.

    This paper is an example technical writing in the Computer Engineering field. It is a guide to instructors on a particular method of teaching technical writing to college students in Computer Engineering. The paper suggests using graphics, tables, and concept maps to assist the learning experience. The author, Professor Roy from University of Aizu in Fukushima, Japan, uses a survey-based study of 25 students to prove the effectiveness of the methods described.
      This is useful as another example of a peer-reviewed journal and a potential learning method. I can implement these techniques into my normal learning method to potentially increase retention.
